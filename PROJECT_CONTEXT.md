# Assistente Financeiro - Contexto do Projeto

## Visão geral

O Assistente Financeiro é o meu primeiro projeto mais estruturado, criado para colocar em prática os conhecimentos adquiridos no primeiro período do curso de Análise e Desenvolvimento de Sistemas. Ele representa uma etapa importante da minha formação, porque reúne conceitos de lógica, linguagem C, organização de código, modularização e controle de versão em um projeto com propósito real.

## Objetivo do sistema

O Assistente Financeiro tem como finalidade ajudar uma pessoa a acompanhar de forma simples suas finanças pessoais. Ele permite registrar dados financeiros básicos e gerar informações úteis para análise de orçamento, dívidas e planejamento.

As funcionalidades principais incluem:

- análise financeira;
- cálculo de saldo restante;
- cálculo de percentual gasto em relação à renda;
- simulação de quitação de dívidas;
- diagnóstico financeiro com dicas;
- cálculo de reserva de emergência;
- definição de metas financeiras;
- estimativa de economia potencial;
- análise básica de despesas;
- persistência de dados em arquivo.

## Evolução do projeto

### Versão 1
- entrada de dados básicos;
- cálculo de saldo simples;
- primeira experiência com console e interação com o usuário.

### Versão 2
- menu interativo com switch/case;
- organização das funcionalidades em opções do programa;
- validações básicas;
- início da navegação mais estruturada no sistema.

### Versão 3
- introdução às estruturas de repetição;
- simulação de pagamento de dívidas mês a mês;
- uso de estruturas condicionais para decisões internas.

### Versão 4
- melhoria na organização do fluxo do programa;
- refinamento das funcionalidades principais;
- melhor experiência de uso no terminal.

### Versão 5
- uso de funções para organizar o código;
- separação de responsabilidades em blocos lógicos.

### Versão 6
- evolução nas regras de negócio;
- inclusão de funcionalidades de reserva, metas e economia potencial;
- análise de despesas com arrays.

### Versão 7
- modularização em múltiplos arquivos;
- uso de structs para armazenar dados financeiros;
- separação em arquivos de cabeçalho e implementação.

### Versão 8
- organização em pastas como src, include, tools e output;
- persistência em arquivo;
- carregamento e salvamento de análises;
- histórico de registros;
- melhor tratamento de entrada de dados.
